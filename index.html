Chromium DWM/MPO rendering bug — minimal reproduction

This page reproduces the rendering glitch affecting Chromium-based apps on Windows 11 24H2 / 25H2 with discrete GPUs. Tracked at Chromium 429859152 and electron/electron#46206.

Trigger steps:

  1. Watch the animation frame counter in the top-left — it should increment ~60 times/sec smoothly.
  2. Press Alt+Tab to switch to a fullscreen application (game, video player, or a maximized window). Or click Fullscreen, then Esc.
  3. Switch back to this tab.
  4. Repeat 2–3 times if needed — the bug doesn't always fire on the first switch.

Bug present: the counter freezes at an old number while the rest of the UI responds, or shows torn/partial frames. Click the page → counter resumes. The bug is in the OS compositor (DWM), not the page.

No bug: counter keeps incrementing throughout, or resumes immediately when window regains focus.
